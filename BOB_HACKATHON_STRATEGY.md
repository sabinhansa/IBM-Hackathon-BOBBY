## Detailed Bob Usage by Workflow

### Workflow 1: Repo Onboarding

**Bob's Role**: Analyze repository structure and generate plain-language onboarding guide

**Bob Mode**: Ask Mode

**Prompt Strategy**:
```
Analyze this repository structure and create a plain-language onboarding guide covering:
1. Project purpose and goals
2. Key directories and their roles
3. Important files (config, entry points, tests)
4. Development workflow and commands
5. First contribution steps for new developers

Output as a markdown document suitable for README.md
```

**Bob Capabilities Used**:
- Repository context understanding
- File structure analysis
- Pattern recognition (common project structures)
- Documentation generation
- Plain-language explanation

**Evidence**:
- Sample onboarding.md generated by Bob
- Task session showing Bob's analysis process
- Screenshots of Bob exploring the repository

## Bob Shell Integration Strategy

### Why Bob Shell Matters

While the static demo showcases Bob's capabilities, **optional Bob Shell integration** demonstrates:
- Real-time workflow automation
- Production-ready tooling
- Practical developer experience
- Scalability beyond demo

### Integration Architecture

```
Developer Workflow
       ↓
BobFlow Web UI (shows what's possible)
       ↓
bobflow-runner CLI (optional local tool)
       ↓
Bob Shell (--auth-method api-key)
       ↓
Repository Analysis
       ↓
Structured Output (markdown, JSON)
       ↓
BobFlow UI Display
```

### Security-First Design

**Critical Requirements**:
1. ✅ API keys only in environment variables
2. ✅ `.bobignore` excludes sensitive files
3. ✅ No auto-approve/yolo mode by default
4. ✅ Clear data handling documentation
5. ✅ User review before applying changes

**Example .bobignore**:
```
.env
.env.*
*.key
*.pem
secrets/
credentials/
node_modules/
.git/
```

### Command Templates

Each workflow has a documented Bob Shell command:

```bash
# Repo Onboarding
bob --auth-method api-key --chat-mode=ask -p "Analyze this repository..."

# Architecture Map
bob --auth-method api-key --chat-mode=plan -p "Create architecture documentation..."

# Test Plan
bob --auth-method api-key --chat-mode=code -p "Generate test plan for changed files..."
```

**Note**: These are documented but not required for the demo. Judges can see the potential without needing to run Bob Shell themselves.
